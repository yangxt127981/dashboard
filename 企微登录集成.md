# 企业微信扫码登录集成方案

## 现有认证机制梳理

| 层 | 现状 |
|---|---|
| Token 类型 | UUID，存内存 `ConcurrentHashMap` |
| 传输方式 | 前端 `localStorage` → 请求头 `Authorization` |
| 用户表 | `id, username, password, role`（无三方绑定字段）|
| 拦截器 | `AuthInterceptor` 从内存 token 查 User |

---

## 企业微信扫码登录完整流程

企业微信使用 **OAuth 2.0 授权码模式**，整体分三步：

```
[前端] 点击"企微扫码"
    → 跳转/嵌入企微授权页（二维码）
    → 用户扫码，企微重定向回 redirect_uri?code=xxx
[后端] 用 code 换 access_token + userid
    → 查本地用户表（按 wxwork_userid）
    → 找不到则自动创建
    → 生成 UUID token 写入内存（复用现有机制）
[前端] 拿到 token/username/role → 正常进入看板
```

---

## 前置条件（需企业微信管理员配置）

1. 登录[企业微信管理后台](https://work.weixin.qq.com/wework_admin/frame)
2. 创建或选择一个**自建应用**，获取以下三个参数：
   - `corpid`：企业 ID
   - `agentid`：应用 ID
   - `corpsecret`：应用密钥
3. 在应用的「企业微信授权登录」中，配置**授权回调域名**（如 `localhost` 或生产域名）

---

## 后端改动（4 处）

### 1. 数据库：User 表新增字段

```sql
ALTER TABLE user ADD COLUMN wxwork_userid VARCHAR(64) NULL COMMENT '企微 UserID';
ALTER TABLE user ADD COLUMN display_name VARCHAR(64) NULL COMMENT '企微真实姓名';
```

### 2. 新增 WxWorkService

负责与企微 API 通信，包含以下方法：

- `getAccessToken()`：用 corpid + corpsecret 换 access_token（**必须后端缓存，有效期 2 小时**）
- `getUserIdByCode(code)`：用授权 code 换企微 userid
- `getUserInfo(userid)`：用 userid 查询姓名、头像等信息

### 3. AuthController 新增回调接口

```
GET /api/auth/wxwork/callback?code=xxx&state=xxx
```

处理逻辑：

1. 校验 state 参数（防 CSRF）
2. 调企微 API，用 code 换取 userid
3. 按 `wxwork_userid` 查本地 `user` 表
4. 查不到 → 自动插入新用户（role 默认 `USER`）
5. 生成 UUID token 写入内存（与现有 login 逻辑完全一致）
6. 返回 `{ token, username, role }`

### 4. application.yml 新增企微配置

```yaml
wxwork:
  corpid: ww...
  agentid: 100...
  corpsecret: xxx
  redirect-uri: http://localhost:5173/wxwork-callback
```

## 前端改动（2 处）

### 1. Login.vue 新增扫码入口

提供两种方案，二选一：

| 方案 | 实现方式 | 用户体验 |
|---|---|---|
| **方案 A（推荐）** | 企微 JS SDK 嵌入二维码 iframe，无页面跳转 | 登录页直接显示二维码，扫码即登录 |
| **方案 B（简单）** | 点击按钮跳转企微授权 URL，扫码后回调 | 点击 → 跳转 → 扫码 → 返回 |

### 2. 新增 /wxwork-callback 路由页面

职责：
1. 读取 URL 的 `code` 和 `state` 参数
2. 调用 `GET /api/auth/wxwork/callback?code=xxx&state=xxx`
3. 拿到 token → 写入 localStorage → 跳转 `/board`（与现有登录成功逻辑完全一致）

## 关键注意事项

| 问题 | 说明 |
|---|---|
| access_token 缓存 | 企微 access_token 每日调用有上限，必须在后端缓存，不能每次重新获取 |
| state 参数防 CSRF | 前端生成随机 state，回调时后端校验，防止伪造请求 |
| 已有账号绑定 | 首次企微登录自动创建 USER 角色账号；若需和现有 admin/user 账号绑定，需额外设计绑定流程 |
| 本地开发调试 | 企微回调域名不支持裸 IP，需使用 `localhost`，或配置 hosts/内网穿透工具 |
| 保留账号密码登录 | 两套登录方式并存，互不影响，现有用户无感知 |
