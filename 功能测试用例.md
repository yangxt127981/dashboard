## 一、登录 / 登出

### TC-AUTH-01 账号密码登录（正常）
- **前置**：未登录状态
- **步骤**：输入 admin / admin123，点击登录
- **预期**：跳转 /board，顶部栏显示「管理员」角色标签和用户名 admin

### TC-AUTH-02 账号密码登录（错误密码）
- **步骤**：输入 admin / 错误密码，点击登录
- **预期**：提示「用户名或密码错误」，停留登录页

### TC-AUTH-03 账号密码登录（空字段）
- **步骤**：用户名/密码为空，点击登录
- **预期**：表单校验提示「请输入用户名」/「请输入密码」

### TC-AUTH-04 USER 角色登录
- **步骤**：输入 meione / 123456 登录
- **预期**：跳转 /board，角色标签显示「普通用户」，无「新增需求」按钮，操作列无编辑/取消/删除

### TC-AUTH-05 IOA 一键登录（正常）
- **前置**：设备已登录 IOA 客户端
- **步骤**：点击「IOA 一键登录」按钮
- **预期**：自动获取 Ticket → 后端验签通过 → 登录成功，进入看板

### TC-AUTH-06 IOA 一键登录（未登录 IOA）
- **前置**：设备未登录 IOA 客户端
- **步骤**：点击「IOA 一键登录」
- **预期**：提示「获取 IOA Ticket 失败，请确认已登录 IOA 客户端」

### TC-AUTH-07 IOA 首次登录自动创建账号
- **前置**：数据库 user 表中不存在该工号
- **步骤**：IOA 登录成功
- **预期**：user 表自动插入一条记录，role = USER，password = IOA_SSO

### TC-AUTH-08 退出登录
- **步骤**：点击顶部栏「退出登录」
- **预期**：清除 localStorage，跳转 /login，再次访问 /board 自动重定向 /login

### TC-AUTH-09 Token 过期 / 直接访问受保护页面
- **步骤**：清空 localStorage 后直接访问 http://localhost:5173/board
- **预期**：自动跳转 /login

## 二、列表 - 筛选与 Tab

### TC-LIST-01 默认加载
- **步骤**：登录后进入看板
- **预期**：Tab 默认「全部」，展示所有需求，分页默认 10 条/页，按优先级排序（紧急→高→中→低）

### TC-LIST-02 需求名称模糊搜索
- **步骤**：筛选区输入部分需求名称关键词，点击「查询」
- **预期**：表格只显示名称包含该关键词的需求

### TC-LIST-03 部门下拉筛选
- **步骤**：选择某个部门，点击「查询」
- **预期**：只显示该部门的需求

### TC-LIST-04 优先级多选筛选
- **步骤**：多选「紧急」「高」，点击「查询」
- **预期**：只显示优先级为紧急或高的需求

### TC-LIST-05 状态多选筛选（Tab=全部时）
- **步骤**：多选「开发中」「测试中」，点击「查询」
- **预期**：只显示对应状态的需求，状态筛选框可用

### TC-LIST-06 状态筛选在非全部 Tab 时禁用
- **步骤**：切换到「进行中」Tab，观察状态筛选框
- **预期**：状态筛选框置灰不可操作

### TC-LIST-07 重置筛选
- **步骤**：填写多个筛选条件后点击「重置」
- **预期**：所有筛选条件清空，表格恢复全量数据

### TC-LIST-08 Tab - 进行中
- **步骤**：点击「进行中」Tab
- **预期**：只显示状态为「设计中/开发中/测试中/已上线」的需求

### TC-LIST-09 Tab - 未开始
- **步骤**：点击「未开始」Tab
- **预期**：只显示状态为「未开始」的需求

### TC-LIST-10 Tab - 已取消
- **步骤**：点击「已取消」Tab
- **预期**：只显示状态为「已取消」的需求，状态标签显示红色

### TC-LIST-11 分页
- **步骤**：数据超过 10 条时，点击第 2 页
- **预期**：正确翻页，显示第 11~20 条

### TC-LIST-12 排序 - 计划完成时间升序
- **步骤**：点击「计划完成」列表头升序箭头
- **预期**：按计划完成时间从早到晚排列

### TC-LIST-13 排序 - 优先级
- **步骤**：点击「优先级」列表头排序
- **预期**：按紧急→高→中→低顺序排列

### TC-LIST-14 刷新按钮
- **步骤**：点击右上角圆形刷新按钮
- **预期**：重新请求数据，列表刷新

## 四、数据统计面板

### TC-STAT-01 展开 / 折叠
- **步骤**：点击「数据统计」标题
- **预期**：面板折叠/展开切换

### TC-STAT-02 三图展示
- **步骤**：展开数据统计面板
- **预期**：显示「部门分布」「优先级分布」「状态分布」三张饼图，有竖向分隔线

### TC-STAT-03 部门图点击联动
- **步骤**：点击部门饼图某个扇区（如「财务部」）

- **步骤**：联动状态下再次点击同一扇区
- **预期**：联动取消，三图恢复全量数据

## 五、新增需求（仅 ADMIN）

### TC-ADD-01 正常新增
- **步骤**：点击「新增需求」，填写需求名称（必填）及其他字段，点击「确定」
- **预期**：弹窗关闭，列表刷新，新需求出现在列表中，操作日志记录「创建」操作

### TC-ADD-02 必填项校验
- **步骤**：需求名称留空，点击「确定」
- **预期**：提示「请输入需求名称」，不提交

### TC-ADD-03 所有字段填写
- **步骤**：填写需求名称、模块、部门（下拉14个）、对接人、产品对接人（下拉）、优先级、状态、计划/实际日期、描述
- **预期**：所有字段正确保存并在详情中展示

### TC-ADD-04 上传附件
- **步骤**：新增需求时上传一张图片（< 10MB）
- **预期**：图片上传成功，新增成功后详情可见附件缩略图

### TC-ADD-05 上传超大文件
- **步骤**：上传 > 10MB 的文件
- **预期**：提示文件过大，拒绝上传

### TC-ADD-06 USER 无新增入口
- **步骤**：USER 账号登录
- **预期**：页面无「新增需求」按钮

- **预期**：弹窗关闭，列表数据更新，操作日志记录「编辑」操作，日志中有字段变更对比

- **步骤**：打开已有附件的需求编辑弹窗
- **预期**：原有附件缩略图正常显示

### TC-EDIT-03 编辑时删除附件
- **步骤**：在编辑弹窗中删除已有附件，提交
- **预期**：附件从详情中消失

### TC-EDIT-04 编辑时新增附件
- **步骤**：在编辑弹窗中上传新图片，提交
- **预期**：新附件出现在详情中

### TC-EDIT-05 详情弹窗快捷跳编辑（ADMIN）
- **步骤**：ADMIN 打开详情弹窗，点击「编辑」快捷入口
- **预期**：详情弹窗关闭，编辑弹窗打开，数据回显正确

- **步骤**：点击「删除」，二次确认弹窗中点击「取消」

## 十一、权限隔离（后端校验）

### TC-PERM-01 USER 调用新增接口
- **步骤**：USER 登录后，通过 Postman 直接调用 `POST /api/requirements`
- **预期**：返回 403 权限不足

### TC-PERM-02 USER 调用编辑接口
- **步骤**：USER 登录后，Postman 调用 `PUT /api/requirements/{id}`
- **预期**：返回 403 权限不足

### TC-PERM-03 无 Token 访问接口
- **步骤**：Postman 不带 Authorization header 调用 `GET /api/requirements`
- **预期**：返回 401 未登录

### TC-PERM-04 Token 失效访问
- **步骤**：后端重启（清空内存 token）后，前端用旧 token 访问
- **预期**：返回 401，前端自动跳转登录页
