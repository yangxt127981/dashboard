# 项目计划看板 - 云服务器部署指南

> 技术栈：Spring Boot 后端 + Vue 3 前端 + MySQL + Nginx

---

## 一、服务器环境准备

```bash
# 安装 Java 17（CentOS / Alibaba Cloud Linux）
yum install java-17-openjdk -y
# Ubuntu
apt install openjdk-17-jdk -y

# 安装 MySQL
yum install mysql-server -y
systemctl start mysqld
systemctl enable mysqld

# 安装 Nginx
yum install nginx -y
systemctl start nginx
systemctl enable nginx
```

---

## 二、初始化数据库

```bash
mysql -u root -p
```

登录后执行：

```sql
CREATE DATABASE dashboard_db DEFAULT CHARACTER SET utf8mb4;
```

退出后导入初始化 SQL：

```bash
mysql -u root -p dashboard_db < /path/to/init.sql
```

> `init.sql` 位于项目 `backend/src/main/resources/sql/init.sql`

---

## 三、打包项目（本地执行）

**打包前端：**

```bash
cd frontend
npm run build
# 产物：frontend/dist/
```

**打包后端：**

```bash
cd backend
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home
mvn clean package -DskipTests
# 产物：backend/target/dashboard-0.0.1-SNAPSHOT.jar
```

---

## 四、上传文件到服务器

```bash
# 上传后端 jar 包
scp backend/target/dashboard-0.0.1-SNAPSHOT.jar root@<服务器IP>:/opt/dashboard/

# 上传前端静态文件
scp -r frontend/dist root@<服务器IP>:/opt/dashboard/frontend/
```

---

## 五、启动后端

```bash
# 在服务器上创建上传目录
mkdir -p /opt/dashboard/uploads

# 进入部署目录
cd /opt/dashboard

# 后台运行，日志写入 app.log
nohup java -jar dashboard-0.0.1-SNAPSHOT.jar \
  --spring.datasource.password=<MySQL密码> \
  > app.log 2>&1 &

# 保存进程 ID，便于后续停止
echo $! > app.pid
```

> 如果 MySQL root 没有密码，去掉 `--spring.datasource.password` 参数即可。

验证后端是否启动成功：

```bash
tail -f /opt/dashboard/app.log
# 看到 "Started DashboardApplication" 表示启动成功
```

## 六、配置 Nginx

创建配置文件 `/etc/nginx/conf.d/dashboard.conf`：

```nginx
server {
    listen 80;
    server_name <你的域名或服务器IP>;

    # 前端静态文件目录
    root /opt/dashboard/frontend;
    index index.html;

    # Vue Router history 模式支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 请求反代到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 上传文件访问
    location /uploads/ {
        proxy_pass http://127.0.0.1:8080;
    }
}
```

检查并重载 Nginx：

```bash
nginx -t
systemctl reload nginx
```

## 七、验证部署

浏览器访问 `http://<服务器IP>`，出现登录页即部署成功。

默认账号：
- 管理员：`admin` / `admin123`
- 普通用户：`user` / `user123`

## 八、常用运维命令

```bash
# 查看后端实时日志
tail -f /opt/dashboard/app.log

# 停止后端
kill $(cat /opt/dashboard/app.pid)

# 重启后端
kill $(cat /opt/dashboard/app.pid)
nohup java -jar /opt/dashboard/dashboard-0.0.1-SNAPSHOT.jar \
  --spring.datasource.password=<MySQL密码> \
  > /opt/dashboard/app.log 2>&1 &
echo $! > /opt/dashboard/app.pid

# 更新部署（重新上传 jar 或 dist 后执行重启）
```

## 九、注意事项

| 事项 | 说明 |
|------|------|
| 安全组 / 防火墙 | 需开放 **80 端口**（HTTPS 需额外开放 443） |
| 8080 端口 | 无需对外开放，仅供 Nginx 内部转发 |
| HTTPS | 推荐配置域名后使用 Let's Encrypt 免费证书：`certbot --nginx` |
| 文件上传持久化 | 上传的图片存储在服务器 `/opt/dashboard/uploads/`，更新 jar 时不要删除此目录 |
| 数据库备份 | 建议定期执行 `mysqldump dashboard_db > backup.sql` |
